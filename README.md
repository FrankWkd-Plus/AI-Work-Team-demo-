# 議 · 硅基群聊 Pro (Siliconflow Group Chat Pro)

> **多模型议会 Pro** —— 极简、现代且功能强大的单文件本地端「多 AI 协同群聊」平台。基于 Siliconflow (硅基流动) 等兼容 OpenAI 格式的 API，让多个 AI 模型在同一个聊天室内各抒己见，支持**工作流串联、查验模式、自由问答、AI 自动编排顺序、思维链展示**等高阶特性。

---

## 🌟 核心特性

- **🎭 多模型席位管理**：支持选择 1~8 个模型席位。自由添加自定义模型，或者一键从服务商 API 拉取全部可用模型库，随心定制专属你的“AI 议会”。
- **🔄 多种对话模式（三种议事规则）**：
  - **工作流模式 (Workflow)**：定义“查找 🔍 -> 规划 📋 -> 输出 📤”三阶段，指派特定模型分别负责，上一步的成果自动作为下一步的上下文，实现无缝链式流。
  - **查验模式 (Review)**：模型依次发言，后发言的模型会自动审视、参考并精简归纳前者的观点，避免无意义的重复，专注于寻找漏洞和做补充。
  - **问答模式 (Debate)**：深度脑暴与碰撞，模型在各自发言时，能够灵活引用、赞同或驳斥其他模型的观点，形成高质量的技术或思想辩论。
- **🎯 科学发言顺序编排**：
  - 支持手动拖拽调整模型发言顺序。
  - 接入 **🤖 AI 编排**，指定一个模型作为主导，自动分析其他模型的性格和 Prompt，智能排出最佳发言序列（例如：思考型模型先分析 $\rightarrow$ 通用模型补全 $\rightarrow$ 总结模型产出）。
  - 支持保存和一键加载发言顺序预设。
- **💭 极致交互与渲染**：
  - **三种渲染模式**：⚡ Fast (超高频实时刷新) | 💭 Think (平滑思考展现) | 🔥 Ultra (低频超平稳批渲染)。
  - **思维链 (Reasoning) 折叠**：完美支持 DeepSeek-R1 / Kimi-K2.5 等推理模型的 `reasoning_content`。思考过程与回答独立折叠，避免冗长的推理打乱阅读节奏。
  - **@ 召唤机制**：在输入框输入 `@` 即可弹出快捷菜单，定向召唤特定席位模型发言。
- **🎨 纯单文件，即开即用**：
  - 基于现代响应式 HTML5 CSS3 构建，内置轻量级 Markdown 渲染器，零编译、零打包、零依赖、无后端。
  - 完美支持 **🌓 昼夜主题交替** 及 **📱 移动端安全区域 (Safe Area) 与视口自适应**。
  - 支持全局数据 **备份导出 (JSON)**、**本地导入** 与一键清空。

## 🛠️ 快速上手

1. **下载本项目**：
   将 `index.html`（或本项目对应的源代码）保存到你本地电脑的任何地方。
2. **本地双击打开**：
   无需配置 Node.js、Python 或任何服务器环境，直接用浏览器（如 Chrome, Safari, Edge）双击打开 `index.html` 即可。
3. **连接 API 服务**：
   - 点击右上角的齿轮图标 **⚙️ 配置中心**。
   - 输入你的**节点端点 (Endpoint)**（默认为硅基流动官方端点：`https://api.siliconflow.cn/v1`，也可无缝对接 DeepSeek、OpenAI、One-API、Local Ollama 等任何标准 API 端点）。
   - 输入你的**授权密钥 (API Key)**。
   - 点击 **🔌 测试连通性**。
4. **添加/导入模型**：
   - 点击 **📡 获取&导入模型**，一键拉取服务商端点下所有可用的模型，并在弹出框中勾选你需要的模型导入。对于推理模型，勾选 **💭 思考** 即可自动激活思维链适配。
   - 也可以通过 **✏️ 模型个性化** 修改各席位机器人的专属 Nickname 以及影响其发言倾向的 System Prompt。
5. **开启你的多模讨论**：
   在底部的输入框输入你的问题，按下 `Enter` 或点击 `➤` 发送，即可静待你的 AI 议会成员开始碰撞火花。

## ⚙️ 架构与数据流动

### 1. 三种对话模式原理

```
【 模式 1：工作流模式 】
[用户输入] ──> (查找阶段：模型A) ──> [查找报告] ──> (规划阶段：模型B) ──> [解决方案] ──> (输出阶段：模型C) ──> [最终完美答案]

【 模式 2：查验模式 】
[用户输入] ──> (模型A 回答) ──> (模型B 结合 模型A 的答案，去重查漏补缺) ──> (模型C 综合评审前两者)

【 模式 3：问答模式 】
[用户输入] ──> (模型A 观点) ──> (模型B 自由引用或反驳 A 观点) ──> (模型C 进行高维度大合流辩论)
```

### 2. 本地数据隔离
本项目不设任何后端服务器，所有的聊天会话（Sessions）、选定的模型配置（Configs）、自定义模型库（Custom Models）均使用 `localStorage` 存储在**你的浏览器本地存储中**。
* 为保证数据安全，建议定期在 **💾 数据管理** 弹窗中点击 **📤 导出备份** 备份你的数据。

## 💡 使用小贴士

* **极致性能调优**：如果导入了大量模型且网络延迟不稳定，建议将渲染模式调整为 `💭 Think` 甚至 `🔥 Ultra`，这会降低 DOM 的重绘频次，有效消除超长流输出时由于浏览器渲染队列拥堵产生的卡顿。
* **自定义中转端点**：本程序完全采用标准 OpenAI 聊天补全接口格式（`/chat/completions` 和 `/models`）。你可以使用它配合任何本地的大语言模型服务（如 Ollama, Llama.cpp, vLLM 等），仅需将端点修改为对应的本地 IP（如 `http://localhost:11434/v1`）即可。
